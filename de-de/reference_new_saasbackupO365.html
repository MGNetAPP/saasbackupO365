    Neue Funktionen und Updates

    Die folgenden neuen Funktionen und Updates wurden dieser Version von NetApp SaaS Backup für Office 365 hinzugefügt

    April 2019

    • SaaS Backup für Office 365 unterstützt jetzt das Löschen von Sicherheitsgruppen.

      Sicherheitsgruppen löschen

    • Freigegebene Postfächer verbrauchen keine Benutzerlizenz mehr.

    März 2019

    • SaaS Backup für Office 365 unterstützt jetzt mehrere Sicherungsspeicherorte in jeder unterstützten Region.
      Sie können jetzt alle verfügbaren Standorte in Ihrer ausgewählten Region als Standort für Ihre Datensicherung auswählen. Es wird empfohlen, den Ort zu wählen, der dem Ort Ihrer Daten geografisch am nächsten liegt. Der von SaaS Backup empfohlene Speicherort wird in der Liste der Optionen als bevorzugt markiert.
      Hinweis : Wenn Sie ein Upgrade von einer Testversion durchführen und einen anderen als den in der Testversion verwendeten Speicherort auswählen, werden Ihre Trail-Daten nicht beibehalten.
      Fertig machen
      Upgrade von einem Testabonnement

    • Sie können jetzt Benutzerlizenzen freigeben und für andere Benutzer verfügbar machen.
      Eine Lizenz freigeben

    Februar 2019

    • SaaS Backup für Office 365 unterstützt jetzt Folgendes:

      • Sicherung und Wiederherstellung von Archivpostfächern.

      • Erweiterte Sicherungs- und Wiederherstellungsstatistiken für Microsoft Office Exchange Online, SharePoint und OneDrive for Business.

    Dezember 2018

    • SaaS Backup für Office 365 kann jetzt über den AppDirect Marketplace und den CANCOM Marketplace erworben werden.

    August 2018

    • Die Benutzeroberfläche wurde für verbesserte Benutzererfahrung und Effizienz neu gestaltet.

    • Die Aufbewahrungsrichtlinien für Daten wurden aktualisiert, sodass die Daten drei Jahre lang aufbewahrt werden können.
      Sicherungsrichtlinien

    Mai 2018

    • NetApp Cloud Control wurde in NetApp SaaS Backup für Office 365 umbenannt.

    • Sie können jetzt Benutzer, Websitesammlungen und Office 365-Gruppen löschen, indem Sie alle zugehörigen Daten vollständig aus SaaS Backup für Office 365 entfernen.
      Bereinigen eines Benutzers, einer Websitesammlung oder einer Office 365-Gruppe

    • SaaS Backup erkennt jetzt sowohl öffentliche als auch private Gruppen für Office 365-Gruppen.

    April 2018

    • SaaS-Sicherung für Office 365 unterstützt jetzt gemeinsam genutzte Postfächer für Microsoft Office Exchange Online.
      Freigegebene Postfächer werden mithilfe eines automatisch erstellten Dienstkontos ermittelt. Wenn Sie vor diesem Update keinen Dienst für Microsoft Office Exchange Online aktiviert haben, wird das automatische Dienstkonto für freigegebene Postfächer von SaaS Backup erstellt, wenn Sie Microsoft Office Exchange Online aktivieren. Wenn Ihr Dienst für Microsoft Office Exchange Online bereits aktiviert ist, müssen Sie SaaS Backup-Berechtigung zum Erstellen des automatischen Dienstkontos erteilen, damit Ihre freigegebenen Postfächer erkannt und gesichert werden können. Erteilen von Berechtigungen zum Aktivieren gemeinsam genutzter Postfächer
      Nachdem Ihr automatisches Dienstkonto erstellt wurde, werden Ihre freigegebenen Postfächer automatisch bei der nächsten geplanten Synchronisierung Ihres Benutzerkontos erkannt. Wenn Ihre freigegebenen Postfächer sofort erkannt werden sollen, können Sie Ihre Benutzerkonten sofort synchronisieren .

    März 2018

    Der Ort, an dem Sie einen Aktivierungscode für eine kostenlose Testversion eingeben, wurde in den Assistenten zum Hinzufügen eines Service Providers verschoben:

    Februar 2018

    • Die auf Vorlagen-ID basierende Filterung ist jetzt für Microsoft SharePoint Online verfügbar.
      Einen benutzerdefinierten Filter erstellen

    • Sie können jetzt das Aktivitätsprotokoll für SaaS Backup für Office 365-Benutzerkonten in eine CSV-Datei herunterladen.
      Aktivitätsprotokoll herunterladen

    • Die Synchronisierung von Benutzerkonten, Standorten und Gruppen zwischen SaaS Backup for Office 365 und Ihrem Dienst ist jetzt standardmäßig aktiviert.

    • Die Aufnahme des Sicherungsversionsverlaufs ist jetzt standardmäßig aktiviert. Die Standardanzahl von Versionen ist 20.
      Sicherungseinstellungen aktualisieren

    Januar 2018

    • Das Aktivitätsprotokoll zeigt jetzt den Namen der Benutzer-ID an, die mit jeder in SaaS Backup für Office 365 durchgeführten Aktion verknüpft ist.

    • Sie können jetzt Ihre Benutzerberechtigungen mit Azure Active Directory manuell in SaaS Backup für Office 365 synchronisieren.

    • Microsoft Exchange Online unterstützt jetzt den Export nach PST für die Wiederherstellung auf Ordnerebene.

    November 2017

    • SaaS Backup für Office 365 unterstützt jetzt Azure Blob als Option für von SaaS Backup bereitgestellten Speicher.

    • SaaS-Sicherung für Office 365 unterstützt jetzt Microsoft Office 365-Gruppen für die Sicherung und Wiederherstellung.
      Microsoft Exchange Online oder Microsoft SharePoint Online müssen aktiviert sein, bevor Sie Microsoft Office 365-Gruppen aktivieren können. Microsoft Office 365-Gruppen können nur durch die Sicherungsrichtlinie der Stufe 3 geschützt werden.

    • Microsoft Exchange Online unterstützt jetzt den Export nach PST für die Wiederherstellung auf Postfachebene.

    Oktober 2017

    • Es können Regeln erstellt werden, mit denen Sie Benutzer basierend auf vordefinierten Kriterien automatisch in eine vorausgewählte Sicherungsstufe verschieben können.
      Sie können Regeln für Microsoft Exchange Online und Microsoft OneDrive for Business erstellen. Sie können keine Regeln für Microsoft SharePoint Online erstellen.
      Regeln erstellen

    Bekannte Probleme und Einschränkungen

    • Teilweise fehlgeschlagener Auftragsstatus für die Wiederherstellung der Websitesammlungsgruppe
      Wenn eine gesamte Websitesammlungsgruppe gelöscht wird, schlägt die Wiederherstellung privater Gruppen in der Sammlung fehl. Dies führt zu einem Wiederherstellungsjobstatus "teilweise fehlgeschlagen". In diesem Fall kann auf die Website nicht über die GUI zugegriffen werden.
