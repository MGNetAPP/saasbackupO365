    Neue Funktionen und Updates

    Die folgenden neuen Funktionen und Updates wurden zu dieser Version von NetApp SaaS Backup für Office 365 hinzugefügt

    Mai 2019

    • SaaS Backup unterstützt jetzt Add-On-Lizenzen.

      Aktivierung einer Add-On-Lizenz

    April 2019

    • SaaS Backup für Office 365 unterstützt jetzt das Löschen von Sicherheitsgruppen.

      Sicherheitsgruppen löschen

    • Freigegebene Postfächer verbrauchen keine Benutzerlizenz mehr.

    März 2019

    • SaaS Backup für Office 365 unterstützt jetzt mehrere Sicherungsspeicherorte in jeder unterstützten Region.

      Sie können nun einen beliebigen verfügbaren Speicherort in Ihrer ausgewählten Region als Standort für Ihre Datensicherung auswählen. Es wird empfohlen, den Standort zu wählen, der dem Standort Ihrer Daten geografisch am nächsten liegt. Der von SaaS Backup empfohlene Speicherort wird in der Liste der Optionen als bevorzugt markiert.
      Hinweis : Wenn Sie ein Upgrade von einer Testversion durchführen und einen anderen Sicherungsspeicherort als den in Ihrer Testversion verwendeten auswählen, werden Ihre Protokolldaten nicht beibehalten.
      Fertig machen
      Upgrade von einem Testabonnement

    • Sie können jetzt Benutzerlizenzen freigeben und für andere Benutzer verfügbar machen.
      Geben Sie eine Lizenz frei

    Februar 2019

    • SaaS Backup für Office 365 unterstützt jetzt Folgendes:

      • Sicherung und Wiederherstellung von Archivpostfächern.

      • Verbesserte Sicherungs- und Wiederherstellungsstatistiken für Microsoft Office Exchange Online, SharePoint und OneDrive for Business.

    Dezember 2018

    • SaaS Backup für Office 365 kann jetzt über den AppDirect Marketplace und den CANCOM Marketplace erworben werden.

    August 2018

    • Die Benutzeroberfläche wurde für eine verbesserte Benutzererfahrung und Effizienz überarbeitet.

    • Die Richtlinien zur Aufbewahrung von Daten wurden aktualisiert, um die Aufbewahrung von Daten für 3 Jahre zu ermöglichen.
      Backup-Richtlinien

    Mai 2018

    • NetApp Cloud Control wurde in NetApp SaaS Backup für Office 365 umbenannt.

    • Sie können jetzt Benutzer, Websitesammlungen und Office 365-Gruppen bereinigen und alle zugehörigen Daten vollständig aus SaaS Backup for Office 365 entfernen.
      Löschen eines Benutzers, einer Websitesammlung oder einer Office 365-Gruppe

    • SaaS Backup erkennt jetzt sowohl öffentliche als auch private Gruppen für Office 365-Gruppen.

    April 2018

    • SaaS Backup für Office 365 unterstützt jetzt freigegebene Postfächer für Microsoft Office Exchange Online.

      Freigegebene Postfächer werden mithilfe eines automatisch erstellten Dienstkontos erkannt. Wenn Sie den Dienst für Microsoft Office Exchange Online vor diesem Update nicht aktiviert haben, wird das automatische Dienstkonto für freigegebene Postfächer von SaaS Backup erstellt, wenn Sie Microsoft Office Exchange Online aktivieren. Wenn Ihr Dienst für Microsoft Office Exchange Online bereits aktiviert ist, müssen Sie SaaS Backup die Berechtigung erteilen, um das automatische Dienstkonto zu erstellen, damit Ihre freigegebenen Postfächer erkannt und gesichert werden können. Erteilen von Berechtigungen zum Aktivieren freigegebener Postfächer
      Nachdem Ihr automatisches Dienstkonto erstellt wurde, werden Ihre freigegebenen Postfächer bei der nächsten geplanten Synchronisierung Ihres Benutzerkontos automatisch erkannt. Wenn Sie möchten, dass Ihre freigegebenen Postfächer sofort erkannt werden, können Sie Ihre Benutzerkonten sofort synchronisieren .

    März 2018

    Der Speicherort, an dem Sie einen Aktivierungscode für eine kostenlose Testversion eingeben, wurde in den Assistenten zum Hinzufügen eines Dienstanbieters verschoben:

    Februar 2018

    • Das Filtern basierend auf der Vorlagen-ID ist jetzt für Microsoft SharePoint Online verfügbar.
      Erstellen eines benutzerdefinierten Filters

    • Sie können jetzt das Aktivitätsprotokoll des SaaS Backup für Office 365-Benutzerkontos in eine CSV-Datei herunterladen.
      Herunterladen des Aktivitätsprotokolls

    • Die Synchronisierung von Benutzerkonten, Standorten und Gruppen zwischen SaaS Backup für Office 365 und Ihrem Dienst ist jetzt standardmäßig aktiviert.

    • Das Einbeziehen des Verlaufs der Sicherungsversion ist jetzt standardmäßig aktiviert. Die Standardanzahl von Versionen ist 20.
      Backup-Einstellungen aktualisieren

    Januar 2018

    • Das Aktivitätsprotokoll zeigt jetzt den Namen der Benutzer-ID an, die jeder in SaaS Backup für Office 365 ausgeführten Aktion zugeordnet ist.

    • Sie können Ihre Benutzerberechtigungen jetzt manuell in SaaS Backup für Office 365 mit Azure Active Directory synchronisieren.

    • Microsoft Exchange Online unterstützt jetzt den Export in PST für die Wiederherstellung auf Ordnerebene.

    November 2017

    • SaaS Backup für Office 365 unterstützt jetzt Azure Blob als Option für den von SaaS Backup bereitgestellten Speicher.

    • SaaS Backup für Office 365 unterstützt jetzt Microsoft Office 365-Gruppen zum Sichern und Wiederherstellen.
      Microsoft Exchange Online oder Microsoft SharePoint Online müssen aktiviert sein, bevor Sie Microsoft Office 365-Gruppen aktivieren können. Microsoft Office 365-Gruppen können nur durch die Sicherungsrichtlinie der Ebene 3 geschützt werden.

    • Microsoft Exchange Online unterstützt jetzt den Export nach PST für die Wiederherstellung auf Postfachebene.

    Oktober 2017

    • Es können Regeln erstellt werden, mit denen Sie Benutzer basierend auf vordefinierten Kriterien automatisch in eine vorausgewählte Sicherungsschicht verschieben können.
      Sie können Regeln für Microsoft Exchange Online und Microsoft OneDrive for Business erstellen. Sie können keine Regeln für Microsoft SharePoint Online erstellen.
      Regeln erstellen

    Bekannte Probleme und Einschränkungen

    • Teilweise fehlgeschlagener Auftragsstatus für die Wiederherstellung der Websitesammlungsgruppe
      Wenn eine gesamte Websitesammlungsgruppe gelöscht wird, schlägt die Wiederherstellung privater Gruppen in der Sammlung fehl, sodass der Status des Wiederherstellungsjobs "teilweise fehlgeschlagen" lautet. In diesem Fall kann über die GUI nicht auf die Website zugegriffen werden.
