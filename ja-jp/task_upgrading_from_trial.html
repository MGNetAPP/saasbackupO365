    バックアップ保存先の種類を変更すると、試用データが失われます。バックアップ先リージョンを変更した場合も、データが失われます。

    ステップ

    1. 左側のナビゲーションペインから[サービス]をクリックします。

    2. Microsoft Office 365のリンクをクリックします。

    3. サブスクリプションの詳細で 、 アップグレードをクリックします。

    4. アップグレードオプションを選択して、[ 次へ ]をクリックします。
      AWS Marketplaceから購入する場合は、[ AWS Marketplace]を選択してください。それ以外の場合は、[ NetAppライセンス]を選択します。

    5. NetAppライセンスを選択した場合

      1. ライセンス情報を入力して、[ Validate Subscription ]をクリックします。
        ライセンス情報の確認が表示されます。

      2. Nextをクリックしてください。
        購読の保存先情報と保存先と地域を含む情報が表示されます。

      3. 保存先の保存先の種類または保存先の保存領域を変更する場合は、次の手順を実行します。

        1. 新しい保存先の保存場所の種類や地域を選択します。
          注 ：ストレージの保存先または保存先の地域を変更した場合、NetApp SaaS Backup for Office 365は試用データを移行しません。続行することに同意する必要があります。

        2. 宛先ストレージタイプを変更する場合は、必要な情報を入力して[ 接続テスト ]をクリックします。

        3. Nextをクリックしてください。

        4. 設定を確認して、[ 保存 ]をクリックします 。

      4. 同じ保存先ストレージタイプと保存先ストレージ領域を維持する場合は、設定を確認して[ 保存 ]をクリックします 。

    6. AWS Marketplaceを選択した場合：
      注：AWSマーケットプレイスを通じたライセンス購読は、SaaS Backupが提供するAmazon S3ストレージのみを使用できます。トライアルサブスクリプションにBYOSストレージオプションを使用した場合、トライアルデータは移行されません。

      1. [ AWS Marketplaceに移動する]リンクをクリックします 。

      2. マーケットプレイスの指示に従ってください。

      3. Microsoft Office 365アカウントでSaaS Backup for Office 365にサインインします。

      4. Microsoft Office 365設定アイコンをクリックします。

      5. サブスクリプションの詳細で、 アクティブ化をクリックします。

        トライアルサブスクリプションにBYOSオプションを使用した場合は、警告が表示されます。 SaaS Backup for Office 365は、試用データを新しい保存先に移行しません。続行することに同意する必要があります。

        注：試用版購読よりもライセンス購読のユーザー数を減らすことを選択した場合、ライセンス購読がアクティブになると、すべてのユーザーアカウントが保護されていない層に移動します。アクティブ化したら、目的のアカウントを手動で保護された層に移動する必要があります。

      6. 保存をクリックします 。
